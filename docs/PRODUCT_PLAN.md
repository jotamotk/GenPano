# GENPANO 产品规划 & Milestone

## 1. 产品定位

**一句话**: GENPANO 是一个 Agent-native 的免费 GEO (Generative Engine Optimization) 监测平台，帮助品牌和营销人员追踪其在 AI 回答中的可见度。通过覆盖 ChatGPT、豆包、DeepSeek 三大主流 AI 引擎，提供全方位的 GEO 监测数据。

**核心差异化**:

1. **Data-First, User-Second**: 平台主动采集全行业数据，用户注册即看到完整的 GEO 监测数据——零等待体验。区别于 Profound (用户填 query 后才采集) 和 Semrush (依赖 opt-in 小样本)。
2. **Agent-Native**: 不只是有 API，而是从架构层面为 AI Agent 设计——MCP Server 支持、结构化数据输出、Agent 可直接消费的报告格式。任何 AI Agent 都能像人类用户一样方便地获取 GENPANO 数据。
3. **智能大样本监测**: GENPANO 基于行业/品牌/产品三维度自动生成模拟真实用户的 query 集合，通过平台级每日全量采集实现更大样本量、更真实的监测覆盖。
4. **三维度深度监测**: 行业趋势 → 品牌健康度 → 产品竞争力，从宏观到微观的完整洞察链路。
5. **免费工具，数据变现**: 工具完全免费使用，通过行业 GEO 数据报告和优化咨询服务盈利。自定义数据采集作为 Phase 2 付费功能。

### GENPANO 的机会窗口

市场处于早期（类比2005年的SEO），碎片化严重，没有明确的品类领导者。Gartner 预测 2026 年搜索引擎流量将下降 25%，AI 查询月处理量已超 25 亿次。免费+Agent-native 的定位可以快速获取用户和开发者生态。

### 3.1 监测指标体系

**PANO Score (综合评分, 0-100)**

核心输出指标，将多维度数据浓缩为一个直观分数:
- **Brand PANO Score**: V(可见度)×30% + R(排名)×25% + S(情感)×20% + C(语境)×15% + A(引用权威)×10%
- **Product PANO Score**: 在品牌维度基础上增加信息准确度和竞争力维度
- **Industry PANO Score**: Share of Voice×50% + 覆盖度×30% + 趋势×20%
- 等级: A+(90-100) / A(80-89) / B(70-79) / C(60-69) / D(40-59) / F(0-39)

**行业维度 (Industry)**
- AI 引擎对该行业 top queries 的回答趋势
- 行业内品牌 share of voice 分布
- 行业热门话题的 AI 回答变化追踪
- 新进入者/退出者检测

**品牌维度 (Brand)**
- 提及率 (Mention Rate): 品牌在相关 query 回答中被提及的比例
- 排名位置 (Position): 品牌在 AI 推荐列表中的排名
- 情感分析 (Sentiment): AI 回答中对品牌描述的正面/负面/中性倾向
- 推荐语境 (Recommendation Context): 品牌被推荐时的场景和条件
- 竞品对比 (Competitor Comparison): 与竞品在同一 query 中的相对表现
- 引用来源追踪 (Source Attribution): AI 引用了哪些来源来支撑品牌相关回答

**产品维度 (Product)**
- 产品提及率和推荐位置
- 产品特性被 AI 提取的准确度
- 产品 vs 竞品的推荐频率对比
- 产品评价情感趋势
- 价格信息准确度监测

### 3.2 智能 Query 生成系统

这是 GENPANO 的核心差异化引擎。采用 **Bottom-Up** 策略：从产品级（最具体、最接近真实用户）向上抽象到品牌级和行业级。

```
输入: 行业 + 品牌(含品类/定位标签) + 产品(含属性标签)
  ↓
Step 1: 产品级 Query (最先生成，最接近真实用户)
  - "小棕瓶和兰蔻小黑瓶哪个好"
  - "小棕瓶适合什么肤质"
  - "预算500内买什么精华液"
  ↓
Step 2: 品牌级 Query (从产品推导，去掉产品名保留品牌)
  - "雅诗兰黛和兰蔻哪个好"
  - "雅诗兰黛口碑怎么样"
  ↓
Step 3: 行业级 Query (从品牌/产品属性抽象，去掉品牌名保留品类+场景)
  - "好用的精华液推荐"、"抗衰精华哪个好"
  - 竞品发现类: "高端精华液品牌推荐" (发现用户未配置的竞品)
  ↓
Step 4: 变体扩展 (对所有层级)
  - 口语化: "有没有好用的..."
  - 场景化: "送女朋友..."、"预算xxx内..."
  - 地域化: (CN) "国内能买到的..."
  ↓
Step 5: 质量控制 & 真实度过滤
  - 去重 & 相似度合并
  - 基于搜索趋势数据校验 query 真实度
  - 剔除过于人造的 query (像SEO关键词堆砌的不要)
  ↓
输出: 高质量 query 集合 (按层级、意图、场景标注)
```

**设计原则**: 行业级 query 的品类词和场景词都从产品/品牌属性中提取，不是凭空让 LLM 编造"美妆行业发展趋势"这类泛 query。真实用户很少这样问 AI。

## 4. 开发方法: Harness Engineering

采用 Harness Engineering 方法——人类驾驭 AI 引擎，从 0→1 构建产品。完整方法论参见 [HARNESS_ENGINEERING.md](./HARNESS_ENGINEERING.md)。

**四阶段生命周期**:
- **Phase 0 产品设计** (Cowork 对话): 人提供 What/Why，AI 输出 How + 结构化文档 → PRD + PLAN + SESSIONS
- **Phase 1 工程构建** (Claude Code Sessions): 每个 Milestone 一个 Session，AI 写全部代码，人 Review + 把关
- **Phase 2 测试验证** (人机协同): AI 写测试 + 修 Bug，人做探索性测试和最终判断
- **Phase 3 上线运营** (持续 Harness): 日常运维 Session + 用户反馈驱动的迭代循环

**人类时间投入估算**: ~30-40h (产品设计 35% + Code Review 25% + 手动测试 20% + 数据审核 10% + 环境部署 10%)
