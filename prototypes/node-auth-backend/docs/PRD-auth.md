# GenPano 注册登录模块 — 产品需求文档 (PRD)

> 版本: v1.0 | 日期: 2026-03-30 | 状态: Draft

---

## 1. 产品概述

### 1.1 背景

GenPano 是一个面向企业用户的 GEO (Generative Engine Optimization) 平台，帮助品牌监测和优化其在 AI 搜索引擎中的可见度。注册登录模块是用户进入平台的第一触点，需要兼顾**安全性**和**易用性**。

### 1.2 目标

- 为企业用户提供安全、快速的注册登录体验
- 通过工作邮箱限制确保用户质量（B2B 产品定位）
- 支持邮箱密码登录和 Google OAuth 两种方式
- 支持中英文双语

### 1.3 成功指标

| 指标 | 目标值 |
|------|--------|
| 注册完成率 | ≥ 70% (从邮箱输入到账户设置完成) |
| 登录成功率 | ≥ 95% |
| 邮箱验证完成率 | ≥ 80% (24h 内) |
| 页面加载时间 | ≤ 2s |

## 2. 目标用户

### 2.1 用户画像

- **角色**: 品牌经理、数字营销人员、SEO/GEO 专家
- **公司规模**: 中大型企业
- **特征**: 使用公司邮箱、对数据安全敏感、可能使用 Google Workspace

### 2.2 用户限制

- **仅允许工作邮箱注册**，屏蔽以下个人邮箱域名:
  - `gmail.com`, `yahoo.com`, `hotmail.com`, `outlook.com`
  - `qq.com`, `163.com`, `126.com`, `sina.com`
  - `icloud.com`, `me.com`, `mail.com`, `protonmail.com`
  - 等其他常见个人邮箱

## 3. 核心用户流程

### 3.1 注册流程

```
用户访问注册页
    │
    ├── 输入工作邮箱 → [后端检查邮箱是否已注册]
    │       │
    │       ├── 已注册 → 提示 "该邮箱已注册" + 跳转登录
    │       │
    │       └── 未注册 → 发送验证邮件 → 跳转「邮件发送成功」页
    │                                       │
    │                                       └── 用户点击邮件中「验证邮箱」按钮
    │                                               │
    │                                               └── 跳转「账户设置」页
    │                                                       │
    │                                                       ├── 填写: 密码、全名、公司名称
    │                                                       ├── 可选: 订阅新闻邮件
    │                                                       │
    │                                                       └── 点击「注册」→ 进入 Dashboard
    │
    └── 点击「使用 Google 继续」→ Google OAuth 授权
            │
            ├── 授权成功 + 新用户 → 跳转「账户设置」页
            └── 授权成功 + 已有用户 → 直接进入 Dashboard
```

### 3.2 登录流程 (两步登录)

```
用户访问登录页
    │
    ├── Step 1: 输入工作邮箱 → [后端检查邮箱]
    │       │
    │       ├── 已注册 → 显示密码输入框 (Step 2)
    │       │       │
    │       │       ├── 密码正确 → JWT → Dashboard
    │       │       └── 密码错误 → 显示 "用户名或密码不正确"
    │       │
    │       └── 未注册 → 显示提示「如果点击 continue，如果账号未注册👉」
    │               │
    │               └── 点击继续 → 跳转注册页
    │
    └── 点击「使用 Google 账号登录」→ Google OAuth
            │
            ├── 已有账户 → JWT → Dashboard
            └── 新用户 → 跳转注册流程
```

### 3.3 忘记密码流程

```
用户点击「忘记密码？」
    │
    └── 跳转忘记密码页
            │
            ├── 输入邮箱 → [后端发送重置邮件]
            │       │
            │       └── 无论邮箱是否存在，均显示「邮件已发送」(防枚举)
            │               │
            │               └── 跳转「重置邮件发送成功」页
            │                       │
            │                       └── 用户点击邮件中「重置密码」按钮
            │                               │
            │                               └── 跳转「重置密码」页
            │                                       │
            │                                       ├── 输入新密码 + 确认密码
            │                                       ├── 密码匹配 → 重置成功
            │                                       │       │
            │                                       │       └── 跳转「重置成功」页 → 返回登录
            │                                       │
            │                                       └── 密码不匹配 → 显示错误
            │
            └── 点击「注册」→ 跳转注册页
```

### 3.4 Google OAuth 流程

```
用户点击 Google 按钮
    │
    └── 重定向到 Google 授权页面
            │
            └── 用户授权 → Google 回调
                    │
                    ├── [后端] 查找是否有关联账户
                    │       │
                    │       ├── 有 → 签发 JWT → 重定向 Dashboard
                    │       └── 无 → 创建账户 → 跳转账户设置页补全信息
                    │
                    └── 用户取消 → 返回登录页
```

## 4. 页面清单与交互说明

### 4.1 登录页 - Step 1 (邮箱输入)

| 元素 | 交互 |
|------|------|
| 工作邮箱输入框 | 实时校验格式，blur 时校验是否为工作邮箱 |
| 「继续」按钮 | 邮箱有效时可点击，调用 check-email API |
| 「Sign Up」链接 | 跳转注册页 |
| 「忘记密码？」链接 | 跳转忘记密码页 |
| Google 按钮 | 发起 Google OAuth |
| 语言切换 | 中英文切换，持久化到 localStorage |

### 4.2 登录页 - Step 2 (密码输入)

| 元素 | 交互 |
|------|------|
| 邮箱显示 (禁用) | 显示 Step 1 输入的邮箱，右侧编辑图标点击返回 Step 1 |
| 密码输入框 | 右侧眼睛图标切换明文/密文 |
| 错误提示 | 空密码: "请输入密码" / 错误密码: "用户名或密码不正确" |
| 「继续」按钮 | 调用 login API |

### 4.3 注册页

| 元素 | 交互 |
|------|------|
| 工作邮箱输入框 | 同登录页校验逻辑 |
| 「注册」按钮 | 调用 register API → 发送验证邮件 |
| 隐私条款文本 | "注册表示我同意 GenPano 隐私政策和条款"，含链接 |
| 「登录」链接 | 跳转登录页 |
| Google 按钮 | 发起 Google OAuth |

### 4.4 账户设置页

| 字段 | 规则 |
|------|------|
| 公司邮箱 * | 预填，只读 (来自验证链接) |
| 密码 * | 最少 8 字符，含大小写和数字；旁有 (i) 图标展示要求 |
| 全名 * | 必填，非空 |
| 公司名称 * | 必填，非空 |
| 订阅 checkbox | 默认勾选 |

### 4.5 邮件发送成功页 (验证邮件)

| 元素 | 交互 |
|------|------|
| 绿色勾号 | 入场 scale-in 动画 |
| 邮箱显示卡片 | 显示已发送的邮箱，编辑图标可返回修改 |
| 步骤提示 | 1. 查看邮箱 2. 点击验证按钮 |
| 「重新发送邮件」按钮 | 调用 resend-verification API, 60s 冷却 |
| 垃圾邮件提醒 | 灰色小字提示 |
| 「查看邮件」链接 | 外部链接到邮箱 Web 客户端 |

### 4.6 忘记密码页

| 元素 | 交互 |
|------|------|
| 邮箱输入框 | 校验工作邮箱格式 |
| 「发送」按钮 | 调用 forgot-password API |
| 「注册」链接 | 跳转注册页 |

### 4.7 重置邮件发送成功页

同 4.5 结构，文案替换为密码重置相关。

### 4.8 重置密码页

| 字段 | 规则 |
|------|------|
| 新密码 * | 同账户设置的密码规则，(i) 图标 |
| 确认密码 * | 必须与新密码一致 |
| 错误提示 | "密码不匹配，请重试。" / "请输入有效密码" |
| 「Reset」按钮 | 调用 reset-password API |

### 4.9 密码重置成功页

| 元素 | 交互 |
|------|------|
| 绿色勾号 | scale-in 动画 |
| 成功文案 | "密码已成功重置，可使用新密码登录" |
| 「返回」按钮 | 跳转登录页 |

## 5. 业务规则

### 5.1 邮箱规则

| 规则 | 描述 |
|------|------|
| 工作邮箱限制 | 仅接受非个人域名邮箱 |
| 前端校验 | 输入时实时校验格式 + blur 校验域名 |
| 后端校验 | 双重校验，防绕过 |
| 大小写不敏感 | 邮箱统一转小写存储 |

### 5.2 密码规则

| 规则 | 描述 |
|------|------|
| 最小长度 | 8 个字符 |
| 复杂度 | 包含大写字母、小写字母、数字 |
| 哈希存储 | bcrypt, salt rounds = 10 |

### 5.3 Token 规则

| Token 类型 | 有效期 | 格式 |
|-----------|--------|------|
| JWT (登录) | 7 天 | 签名 token, 含 userId/email |
| 验证邮箱 token | 24 小时 | UUID v4 |
| 密码重置 token | 1 小时 | UUID v4 |

### 5.4 速率限制

| 端点 | 限制 |
|------|------|
| `/auth/login` | 10 次/分钟/IP |
| `/auth/register` | 10 次/分钟/IP |
| `/auth/forgot-password` | 5 次/分钟/IP |
| `/auth/resend-verification` | 3 次/分钟/IP |
| 其他 auth 端点 | 20 次/分钟/IP |

### 5.5 安全规则

| 规则 | 描述 |
|------|------|
| 防枚举 | forgot-password 不论邮箱是否存在均返回成功 |
| CORS | 仅允许前端域名 |
| JWT 验证 | 每个受保护路由通过 middleware 验证 |
| HTTPS | 生产环境强制 HTTPS |

## 6. 邮件设计

### 6.1 验证邮件

- **主题**: "Verify your GenPano account / 验证您的 GenPano 账号"
- **发件人**: "GenPano" <noreply@genpano.com>
- **内容**: 欢迎语 + 验证按钮 (链接到 `/setup?token={token}`) + 引导区
- **过期**: 24 小时

### 6.2 重置密码邮件

- **主题**: "Reset your GenPano password / 重置您的 GenPano 密码"
- **发件人**: 同上
- **内容**: 提醒语 + 重置按钮 (链接到 `/reset-password?token={token}`) + 安全提示
- **过期**: 1 小时

## 7. 非功能需求

### 7.1 安全性

- [x] 密码 bcrypt 哈希存储
- [x] JWT 签名验证
- [x] 速率限制
- [x] 防邮箱枚举
- [ ] CSRF 防护 (JWT-only 架构，无 cookie 依赖)
- [ ] Content Security Policy headers
- [ ] HTTPS 强制 (生产环境)

### 7.2 国际化 (i18n)

- 支持中文 (默认) 和英文
- 语言偏好持久化到 localStorage
- 右上角语言切换器
- 邮件模板双语

### 7.3 响应式

- Desktop (≥ 1024px): 左右分栏
- Tablet (768-1023px): 左面板缩小
- Mobile (< 768px): 纯表单，无左面板

### 7.4 性能

- 首屏加载 ≤ 2s
- 粒子动画不影响交互帧率 (≥ 60fps)
- 代码分割: auth 页面懒加载

### 7.5 可访问性

- 所有输入框有 label 关联
- 错误提示有 aria-live 属性
- 键盘导航支持
- 颜色对比度 ≥ 4.5:1

## 9. API 端点清单

| 方法 | 路径 | 描述 |
|------|------|------|
| POST | `/api/auth/register` | 注册 (发送验证邮件) |
| POST | `/api/auth/login` | 邮箱密码登录 |
| POST | `/api/auth/check-email` | 检查邮箱是否已注册 |
| POST | `/api/auth/setup` | 完成账户设置 |
| POST | `/api/auth/forgot-password` | 发送密码重置邮件 |
| POST | `/api/auth/reset-password` | 重置密码 |
| GET | `/api/auth/resend-verification` | 重新发送验证邮件 |
| GET | `/api/auth/me` | 获取当前用户信息 |
| GET | `/api/auth/google` | 发起 Google OAuth |
| GET | `/api/auth/google/callback` | Google OAuth 回调 |
